# 모바일 alert UI 오류 해결 및 거주지역 통일성 확립

## 1. 배경 (Context)
* **모바일 기기 UX 오류**: 상담 신청(전화 상담, 방문 상담, 온라인 게시판 상담) 완료 시 브라우저 기본 `alert()` 창이 특정 모바일 기기 및 인앱 브라우저에서 화면 아래로 잘려 나가는 문제가 발생하였습니다. 사용자가 확인 버튼을 누르지 못해 화면이 굳거나 사이트를 나갈 수 없는 심각한 이탈 요인(UX 결함)이 확인되었습니다.
* **거주지역 데이터 불일치**: 각 상담 신청 폼 및 1분 자가진단 폼에서 거주지역을 수집하는 옵션이 제각기 다르게 정의되어 있어(`서울/경기/인천/기타`, `서울/경기/기타`, 전국 단위 등), 통일된 데이터를 관리자 대시보드와 DB에 적재하기 어려운 문제가 있었습니다.

## 2. 의사결정 (Decisions)
1. **커스텀 성공 화면(Success View) 도입**: 데이터 전송 성공 시 띄우던 기존 브라우저 네이티브 `alert()`를 제거하고, `isSubmitted` 리액트 상태값을 활용하여 기존 폼 자리에 즉각적으로 성공 완료 그래픽(Success UI) 및 이동 버튼을 렌더링하도록 변경했습니다.
2. **거주지역 항목의 전국 단위 일치**:
   * 방문 상담의 전국 17개 광역시/도(서울, 경기, 인천, 부산, 대구, 대전, 광주, 울산, 세종, 강원, 충북, 충남, 전북, 전남, 경북, 경남, 제주) 드롭다운을 표준 포맷으로 채택하였습니다.
   * 실시간 전화 상담 및 온라인 게시판 상담의 선택 옵션을 17개 지역으로 확장 및 `required` 처리를 완료하였습니다.
   * **1분 자가진단 페이지**에 누락되어 있던 거주지역 필드를 Step 1 폼 내에 신규 배치하고 `/api/leads` 데이터베이스 필드에 연동하여 데이터 흐름을 완성시켰습니다.

## 3. 파급 효과 및 결과 (Consequences)
* 모바일 기기에서 확인 버튼의 뷰포트 유실 우려가 완전히 사라졌으며, 신청 완료 후 홈 및 목록으로 복귀하는 동선이 매우 자연스러워졌습니다.
* 데이터베이스(Prisma Model)의 `location` 항목에 모든 유입 소스(전화, 방문, 게시판, 자가진단)가 동일한 17개 지역 문자열 규격으로 적재되어, 통계 분석 및 데이터 관리가 매우 용이해졌습니다.
* `npm run build`를 통한 TypeScript 빌드 안정성이 완벽히 검증되었습니다.
