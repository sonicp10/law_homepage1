# 2026-05-15 Mobile Responsive UI Optimization

## Context & Motivation
사용자가 모바일 디바이스(스마트폰 등 좁은 폭의 화면)에서 웹사이트를 열람할 때 텍스트가 잘리거나 강제로 줄바꿈되고, 입력 폼이나 게시판 테이블 영역이 찌그러지는 등 다양한 UI 레이아웃 깨짐 현상이 보고되었습니다. 이를 해결하여 모바일 환경에서도 최적의 사용자 경험을 제공하고 데스크탑과 동일하게 세련된 디자인을 유지하기 위해 사이트 전반의 반응형 UI 대대적 점검 및 교정을 진행했습니다. 또한 하드코딩 되어있던 방문 상담 달력을 실제 작동하는 기능으로 활성화했습니다.

## Decisions

### 1. 텍스트 강제 줄바꿈(Break-keep) 및 여백 최적화
- **적용 대상:** `SubPageBanner.tsx`, `PostsPageClient.tsx`, 성공사례 목록(`s_story/bankruptcy/page.tsx`, `s_story/review/page.tsx`, `s_story/rehab/page.tsx`) 등.
- **결정 사항:** 한국어 단어가 너비 부족으로 인해 어절 중간에서 잘리는 것("면/책", "이야/기", "목소/리")을 방지하기 위해 `break-keep` 속성과 `whitespace-nowrap`을 주요 텍스트와 버튼에 적용했습니다. 
- **이유:** 시각적 안정감을 주어 법무사 사이트로서의 신뢰감을 높이고 정보 전달력을 명확히 하기 위함입니다.

### 2. 모바일 친화적 입력 폼 구조 (Stacking) 적용
- **적용 대상:** `intro/map/page.tsx`, `qna/visit/page.tsx`, `rehab/step/page.tsx`, `bankruptcy/step/page.tsx`.
- **결정 사항:** 기존 2열 구조(Grid-cols-2)로 억지로 끼워져 있어 찌그러지던 폼 필드와 버튼들을 모바일 화면에서는 1열(flex-col 또는 grid-cols-1)로 상하 배치되도록 교정했습니다.
- **이유:** 터치 친화적인 너비를 확보하고, 좁은 화면에서 가시성을 향상시키기 위한 반응형 웹 표준 대응입니다.

### 3. 방문 상담 예약 달력(Calendar) 기능 활성화
- **적용 대상:** `qna/visit/page.tsx`
- **결정 사항:** 목업 형태였던 방문 달력을 React useState를 활용하여 월별 이동, 일자 선택, 과거 날짜 비활성화(Disabled) 로직이 포함된 실제 기능으로 구현했습니다.
- **이유:** 실제 고객이 정상적으로 예약 접수 기능을 이용할 수 있게 하기 위함입니다.

### 4. 게시판 테이블 모바일 최적화
- **적용 대상:** `qna/board/page.tsx`
- **결정 사항:** 과도하게 넓었던 테이블 여백(Padding)을 모바일 기기에 맞게 대폭 줄이고(`px-10` -> `px-4 sm:px-10`), `min-w-[700px]`을 `min-w-[500px]` 수준으로 낮추어 화면 밖으로 크게 벗어나는 스크롤 현상을 개선했습니다.
- **이유:** 사용자 편의성을 해치는 좌우 스크롤을 최소화하면서도 게시판의 모든 데이터를 무리 없이 제공하기 위함입니다.

## Status
- **Status:** Approved & Merged
- **Implementation:** 반영 및 Vercel 배포 완료 (`main` 브랜치 동기화됨).

## Consequences
- 사이트 전체에 걸쳐 좁은 모바일 화면에서도 UI 찌그러짐 현상이 모두 제거되어 안정적인 반응형 레이아웃이 보장됩니다.
- 모바일에서 방문 상담 날짜를 실제 달력 인터페이스를 통해 편리하게 예약할 수 있게 되었습니다.
