# 의사결정 기록: 네비게이션 및 전환 최적화 (2026-04-14)

## 1. 개요
법률사무소 웹사이트의 의뢰인 전환율을 높이기 위해 자가진단 시스템을 내부화하고, 접근성이 뛰어난 하이엔드 상담 위젯을 구현함.

## 2. 주요 의사결정 및 구현 사항

### A. 3분 자가진단 시스템 (/qna/diagnosis)
- **배경**: 기존 외부 Tally 폼의 이질감을 없애고 사이트 내에서 완결성 있는 진단 경험 제공 필요.
- **결과**: React 기반 3단계 멀티스텝 폼 구현. 10여 개의 핵심 문항으로 압축하여 3분 이내 완료 가능하도록 최적화.

### B. 프리미엄 드래그 위젯 (FloatingConsultForm)
- **디자인 진화**:
  1. Galaxy S7 스타일의 클래식 프레임 (초기)
  2. 홈페이지와 조화로운 '파스텔 웜' 화이트 테마
  3. 시인성 극대화를 위한 '다크 파스텔 + 화이트 입력창' 테마
  4. 최종: **샴페인 골드 프레임 + 딥 네이비 스크린**의 하이엔드 하이브리드 디자인.
- **기능**: '광역 드래그' 기능을 추가하여 사용자가 원하는 위치로 자유롭게 이동 가능. 로컬 스토리지를 활용해 위치 정보 유지.

### C. 네비게이션 UX 개선
- **탭 정렬**: 모든 서브페이지 탭의 간격을 동일하게 확대하여 클릭 편의성 및 가독성 개선.
- **경로 통합**: 자가진단과 1:1 상담 경로를 `/qna` 하위로 일원화하여 관리 효율성 증대.

## 3. 기술적 특징
- **DB 연동**: Prisma Lead 모델에 `extraInfo` JSON 필드를 추가하여 정형화되지 않은 자가진단 데이터를 모두 수집 가능하도록 설계.
- **반응형**: 데스크탑에서는 하이엔드 프레임을, 모바일에서는 콤팩트한 원형 버튼으로 자동 전환되는 반응형 로직 적용.

## 4. 향후 과제
- 수집된 리드 데이터의 어드민 대시보드 시각화 및 알림 기능 추가 필요.
- 사용자 드래그 패턴 분석을 통한 최적의 위젯 초기 위치 재검토.
