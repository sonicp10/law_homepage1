# Decision: 관리자 인증 시스템 전면 개편 및 레이아웃 충돌 해결

## Context
기존의 단순 PIN 번호 방식은 다중 관리자 운영과 권한 세분화에 한계가 있었음. 또한, 관리자 페이지 진입 시 일반 홈페이지의 UI 요소(헤더, 푸터, 플로팅 메뉴)가 겹쳐서 나타나는 레이아웃 충돌 문제가 발생함.

## Proposed Changes

### 1. 보안 및 권한 시스템 (RBAC)
- **DB 기반 인증**: `bcryptjs`와 `jose`를 사용하여 이메일/패스워드 기반의 JWT 인증 시스템 구축.
- **역할 기반 제어**: `SUPERADMIN` 및 `ADMIN` 역할 도입. 각 관리자별로 게시글 관리, 상담 관리, 관리자 관리 권한을 개별 부여 가능.
- **세션 정책**: 보안 강화를 위해 브라우저 종료 시 자동으로 로그아웃되는 '세션 쿠키' 정책 적용 (`admin_session_v1`). 기존의 영구 쿠키를 무효화하기 위해 쿠키 ID 변경.

### 2. 관리자 UI 및 레이아웃
- **LayoutIsolation**: `PublicLayoutWrapper`를 통해 `/admin` 경로에서는 일반 헤더/푸터/플로팅 위젯이 렌더링되지 않도록 수정.
- **Admin Dashboard**: 관리자용 사이드바 레이아웃 최적화 및 권한에 따른 메뉴 필터링 구현.

## Consequences
- 관리자 보안성 대폭 향상.
- 실무자별 계정 생성을 통한 체계적 운영 가능.
- 관리자 페이지의 UI가 깔끔하게 분리되어 작업 효율성 증대.

## Verification
- 서버/클라이언트 세션 동기화 확인 완료.
- 권한별 API 접근 제어 로직 검증 완료.
