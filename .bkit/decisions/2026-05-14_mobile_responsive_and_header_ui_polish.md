# 결정 기록: 모바일 반응형 개선 및 데스크톱 헤더 UI 정렬

**날짜:** 2026-05-14
**상태:** 완료
**참여자:** Antigravity (AI), 사용자

## 배경
Vercel 배포 후 모바일 기기에서 홈페이지의 메뉴가 가로로 나열되어 화면이 잘리는 현상(Overflow)과 데스크톱 메가 메뉴의 정렬 불일치 문제가 보고되었습니다. 이를 해결하여 모든 기기에서 전문적인 법률 서비스를 제공할 수 있는 UI/UX를 구축하고자 합니다.

## 결정 사항

### 1. 모바일 반응형 헤더 (Mobile Hamburger Menu)
- **문제**: 가로 너비가 좁은 모바일에서 데스크톱용 메뉴가 그대로 노출되어 가로 스크롤 발생.
- **해결**: 
  - `lg:hidden` 속성을 이용한 모바일 전용 햄버거 버튼(`≡`) 도입.
  - 우측 사이드바 형태의 모바일 전용 메뉴 패널 구현.
  - `sticky` 설정을 통해 스크롤 시에도 상단 접근성 유지.

### 2. 메인 페이지 Hero Section 및 본문 최적화
- **문제**: `text-6xl` 등 큰 폰트 크기와 넓은 패딩으로 인해 모바일에서 텍스트가 깨짐.
- **해결**:
  - `text-5xl md:text-8xl` 등 반응형 텍스트 크기 적용.
  - 버튼 배치를 `flex-col sm:flex-row`로 변경하여 모바일 가독성 확보.
  - 상담 신청 폼(`RequestSection`)의 2열 그리드를 모바일에서 1열로 자동 전환.

### 3. 데스크톱 메가 메뉴 정렬 및 위치 조정
- **문제**: 메인 메뉴 항목과 하단에 나타나는 서브 메뉴 열의 세로 정렬이 일치하지 않음.
- **해결**:
  - 메뉴 아이템에 고정 너비(`w-[120px]`)를 부여하고 `gap`을 일치시켜 세로 정렬 동기화.
  - 좌측 타이틀 영역의 여백(`ml-16`)을 조정하여 전체 메뉴 콘텐츠를 우측으로 시각적 균형에 맞게 이동.

## 결과 및 향후 과제
- **결과**: PC와 모바일 모두에서 화면 잘림 없이 깔끔한 레이아웃이 구현됨.
- **향후 과제**: 
  - Vercel 배포 완료 후 실제 모바일 기기에서 최종 사용자 테스트 수행.
  - 사용자 피드백에 따른 세부 간격(Padding/Margin) 추가 미세 조정.
