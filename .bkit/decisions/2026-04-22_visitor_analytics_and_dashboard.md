# [Decision] 2026-04-22 방문자 통계 분석 및 성과 대시보드 구축

## 개요
홈페이지의 방문자 데이터(Traffic)를 수집하고, 이를 관리자 페이지에서 실시간 성과 지표(Conversion)로 시각화하여 데이터 기반의 의사결정을 지원하는 분석 시스템을 구축함.

## 주요 구현 사항

### 1. 데이터 수집 체계 (Tracking)
- **Prisma Analytics 모델**: `path`, `referrer`, `ip`, `userAgent` 등을 기록하는 모델링 (Supabase PostgreSQL 사용).
- **자동 추적기**: `AnalyticsTracker` 클라이언트 컴포넌트를 루트 레이아웃에 배치하여 페이지 이동 시마다 비동기로 트래킹 API 호출.
- **필터링**: 관리자 본인의 활동이 통계에 영향을 주지 않도록 `/admin` 경로 시작 페이지는 트래킹 대상에서 제외.

### 2. 통계 분석 API (`/api/admin/analytics/stats`)
- **멀티 피어리드 지원**: 일간, 주간, 월간, 연간 프리셋 및 사용자 정의 기간(Custom Range) 필터링 지원.
- **실시간 지표 집계**: `Lead`, `Consultation`, `BoardQna` 세 가지 테이블의 데이터를 현재 기간에 맞춰 실시간 카운트.
- **지능형 증감율(Trend) 계산**: 현재 선택된 기간과 직전 동일 기간의 데이터를 비교하여 성과 변화율(%)을 수치화.

### 3. 관리자 대시보드 UI
- **시각화**: `recharts` 라이브러리를 사용하여 방문자 추이를 AreaChart로 직관적으로 표시.
- **성과 카드**: 총 페이지뷰, 순 방문자, 평균 유입률, 전환 신청 건수를 카드 형태로 상단 배치.
- **인기 콘텐츠 분석**: 가장 많이 조회된 페이지 TOP 10 및 유입 소스(Referrer) 순위 제공.

## 기술적 의사결정
- **성능**: DB 부하를 줄이기 위해 Prisma의 `groupBy`와 `count` 함수를 활용하여 서버 사이드에서 집계 후 간소화된 데이터만 반환.
- **정확도**: 중복 IP 기반의 `uniqueVisitors` 로직을 도입하여 허수 방문자 필터링.
- **UI/UX**: `lucide-react` 아이콘과 파스텔톤 컬러 시스템을 적용하여 법률 사무소의 신뢰감 있는 브랜드 이미지 유지.

## 향후 과제
- 방문자 수가 급증할 경우를 대비하여 통계 API 결과에 대한 캐싱(Redis 등) 도입 검토.
- 특정 마케팅 채널(UTM 파라미터 등)에 대한 성과 추적 기능 고도화.
