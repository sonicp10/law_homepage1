# 의사결정: 방문 상담 신청 폼 내 예약시간 및 거주지역 드롭다운 커스텀 컴포넌트화 개편

## 컨텍스트 (Context)
기존에 방문 상담 신청 페이지(`VisitConsultPage`)에서 제공되던 **'예약시간'** 및 **'거주지역'** 선택 요소는 HTML5 표준인 네이티브 `<select>` 엘리먼트를 사용하고 있었습니다. 
비주얼 가이드라인을 위해 커스텀 Chevron Arrow 아이콘을 추가하고 포인터 이벤트를 조율했음에도 불구하고, 일부 모바일 환경 및 특정 브라우저 환경에서 네이티브 드롭다운 메뉴가 열린 상태에서 선택지 항목(Option)을 탭하거나 클릭할 때 상태 값이 정상적으로 업데이트되지 않는 치명적인 사용자 상호작용 이슈가 발생하였습니다.

## 분석된 원인 (Problem Analysis)
1. **OS 및 브라우저 네이티브 이벤트 비호환**:
   일부 브라우저(특히 모바일 환경 및 윈도우 크롬 등)에서 스타일이 심하게 적용된 네이티브 `<select>` 요소와 포인터 이벤트 제어가 겹치면서, 클릭 이벤트가 React의 가상 돔(Virtual DOM) 이벤트 시스템에 온전히 전달되지 않고 차단되는 오동작이 발생했습니다.
2. **비주얼 제약**:
   네이티브 드롭다운은 운영체제(OS)의 기본 렌더링 스타일을 따르기 때문에, 웹사이트 전체의 프리미엄 디자인 콘셉트(Watercolor, 골드 톤 테마 등)와 완벽한 조화를 이루지 못하고 이질감을 주었습니다.

## 결정된 솔루션 (Proposed Solution)
*   **완전 제어형 커스텀 드롭다운 (Fully Controlled Custom Dropdown) 도입**:
    네이티브 HTML `<select>`와 `<option>` 방식을 완전히 폐기하고, React 상태로 열림/닫힘 및 값을 제어하는 완전한 커스텀 UI 컴포넌트로 개편을 단행했습니다.
*   **구현된 상세 기능**:
    - **통합 상태 관리**: `isTimeDropdownOpen`, `isLocationDropdownOpen` 상태를 선언하여 상호 교차 닫힘을 지원합니다 (예: 하나가 열리면 다른 하나는 자동 닫힘).
    - **외곽 영역 클릭 감지 (Click-Away Listener)**: `useEffect`를 통해 전역 마우스 이벤트를 구독하여, 열려 있는 드롭다운 외부의 다른 영역을 클릭하면 드롭다운이 자연스럽게 닫히도록 마감 처리했습니다.
    - **고급 프런트엔드 스타일링**:
      - 드롭다운이 열릴 때 Chevron 다운 아이콘이 부드럽게 180도 회전하는 트랜지션을 제공합니다.
      - 리스트 아이템 마우스 호버(Hover) 시 세련된 회색 톤(`hover:bg-gray-50`) 배경이 나타나고, 현재 선택된 아이템은 테마 골드 색상(`bg-[#A67C52] text-white`)으로 또렷하게 강조 표시됩니다.
    - **자체 폼 제출 유효성 검증**:
      - 기존 HTML5 네이티브 validation(`required`)의 한계를 보완하기 위해, `handleSubmit` 상단에 명시적인 값 검증 코드를 탑재하여 사용자가 시간이나 지역을 누락한 채 예약을 시도할 경우 직관적인 알림 메시지를 띄우도록 보강했습니다.
